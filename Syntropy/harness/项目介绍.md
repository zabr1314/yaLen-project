## What is Harness?

> **Harness** 不是框架，而是一种**治理哲学**——它意味着给无序的 Agent 群落装上缰绳与鞍具，让并发的智能体在安全边界内自主奔腾，同时让人类始终握有最终的缰绳。

在 Syntropy 中，Harness 体现为三层：**调度 Harness**（结构化 ACP 协议 + 并行批量调度）、**记忆 Harness**（RRF 混合检索 + LLM 主动捕获）、**治理 Harness**（风险分级工具 + 御批审批流 + 全链路 Tracer）。

我们不做又一个黑盒 AutoGPT——我们要让 8 个 AI 官员在像素沙盘上各安其位，让每一次工具调用都经风险权衡，让每一次人机协同都有迹可循。

## ✨ 关键特性

- 🏛️ **朝廷隐喻可视化** — 8 位 AI 官员（丞相/史官/六部）在 Phaser 3 像素沙盘上实时动画，Agent 状态机（`IDLE → THINKING → ACTING → WAITING → SLEEPING`）映射为角色行为
- 🔄 **结构化 ACP 调度协议** — `Kernel.dispatch()` 直接传递结构化字段，消除传统 Multi-Agent 的"自然语言转译损耗"，Sub-Agent 继承派生深度，防无限递归
- ⚡ **并行批量调度** — `call_officials` 技能基于 `Promise.allSettled` 实现真正的并发执行，N 倍理论加速，单点失败不阻塞全局
- 🧠 **RRF 混合记忆检索** — SQLite FTS5（BM25 关键词）+ BGE Vector（语义向量）双路召回，Reciprocal Rank Fusion 加权排序，精确与模糊兼得
- 💾 **LLM 主动记忆捕获** — Agent 自主判断信息价值，主动调用 `save_memory` 工具写入长期记忆，零规则维护，语义分类（personal/preference/project/decision）
- 🛡️ **风险分级 + 御批审批** — 工具按 Low/Medium/High 三级风险评估，中高风险触发 `WAITING_FOR_HUMAN` 状态，前端弹出御批弹窗，批准/驳回反馈注入上下文继续执行
- 🔍 **全链路 Tracer 可观测性** — 每次指令生成唯一 `traceId`，8 种结构化诊断事件（`agent.turn` / `tool.call` / `dispatch` / `approval.wait` / `agent.stuck` 等），日志脱敏 + 3 分钟卡死检测
- 🔧 **配置热更新 & 知识库管理** — 运行时动态调整 Agent 的 System Prompt / Model / Skill，专属 workspace 支持文件上传与自动向量化索引

## 🏗️ 架构速览

Syntropy 采用 **Kernel-Registry-Bridge** 三层架构：

- **Kernel（调度层）**：负责 Agent 注册、消息路由（`dispatch`/`handleCommand`）、会话管理与审批流恢复，完全解耦于传输层
- **Registry（智能体层）**：8 位官员各持独立的状态机、记忆上下文与技能集，通过 `EventBus` 内部 Pub/Sub 解耦通信
- **Bridge（可视化层）**：Zustand 作为单一事实来源，毫秒级同步 React UI 与 Phaser 3 游戏渲染，Socket.io 承载实时双向事件流

底层基础设施包括：LLM Service（DeepSeek / GPT-4o）、MemoryManager（SQLite + FTS5 + Vector）、Tracer（链路追踪）、SessionStore（SQLite 持久化）。

## 🚀 快速开始

```bash
# 1. 安装依赖
pnpm install

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 填入你的 DeepSeek / OpenAI API Key

# 3. 启动后端（Port 3001）
node server/index.js

# 4. 启动前端（Port 5173）
npm run dev

# 5. 访问 http://localhost:5173，开始上朝 🏛️
```

> **前置要求**: Node.js >= 18，推荐 pnpm
