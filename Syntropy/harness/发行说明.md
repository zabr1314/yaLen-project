## 🎯 这一版的核心叙事

Syntropy v2.5 不只是一个功能更新，而是一次**叙事重构**。

我们从"古代朝廷隐喻的可视化多 Agent 系统"进化为 **"Agent Harness —— 智能体的缰绳与连接器"**。这不是包装，而是对我们现有架构设计哲学的正名：

> 真正的挑战不是单个模型有多强，而是你怎么把它 Harness 起来，让它从能聊天变成能稳定干活、持续交付。

## 🏗️ 架构成熟度回顾（v2.0 → v2.5）

| 维度 | v2.0 | v2.4 | v2.5 (本次) |
|:---|:---|:---|:---|
| **Agent 协作** | 串行 dispatch | 并行 `Promise.allSettled` + ACP 协议 | + Harness 叙事包装 |
| **上下文管理** | 无 | Token 预算 + 安全压缩 | + Context Reset 设计思路 |
| **记忆系统** | 无 | RRF 混合检索 + 主动捕获 | + 记忆即身份哲学 |
| **可观测性** | 基础日志 | Tracer + traceId + 8 种诊断事件 | + 缰绳张力传感器隐喻 |
| **人机协同** | 无 | 风险分级 + 御批弹窗 | + 约束释放创造力论述 |
| **文档体系** | README | README + 架构文档 + 进度追踪 | **+ Harness 五件套** |

## 🎭 Demo 场景：修建江南水库

推荐用此场景进行 4 分钟现场演示：

1. **皇帝下旨** → 丞相接收，状态变为 THINKING
2. **Plan Preview** → 前端展示执行计划树
3. **并行调度** → 户部/工部/兵部同时出动（沙盘寻路动画）
4. **知识检索** → 工部检索《水利工程纪要》，Memory Debugger 展示 RRF 权重
5. **风险拦截** → 兵部"征用民夫 5000 人"触发 HIGH 风险，弹窗御批
6. **人工驳回** → 兵部修正为"招募自愿工匠"，闭环迭代
7. **结果聚合** → 丞相汇总三部报告，诏令标记 Completed

完整脚本见 [**演示脚本.md**](./演示脚本.md)。

## 🔗 与 Anthropic Harness Design 的呼应

本次发行深度参考了 Anthropic 官方 Harness 工程实践（[原文](https://www.anthropic.com/engineering/harness-design-long-running-apps)），并在以下维度形成互补：

| Anthropic 洞察 | Syntropy 的对应实现 |
|:---|:---|
| Generator-Evaluator 分离 | 刑部作为独立 Evaluator 角色（可扩展） |
| Sprint Contract | 丞相调度前的"响应契约"协商 |
| Context Reset + Handoff | `onSleep` 记忆压缩 → 可扩展为结构化 Handoff Artifact |
| Prompt Caching | ContextManager 的 System Prompt 复用机制 |
| 模型演进 → Harness 简化 | 配置热更新 + 运行时动态调整 |

## 📦 快速开始

```bash
# 1. 安装依赖
pnpm install

# 2. 配置环境
cp .env.example .env
# 编辑 .env，填入 DeepSeek 或 OpenAI API Key

# 3. 启动后端（Port 3001）
node server/index.js

# 4. 启动前端（Port 5173）
npm run dev

# 5. 打开 http://localhost:5173，开始"上朝"
```
