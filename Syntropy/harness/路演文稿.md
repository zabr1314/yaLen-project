# Syntropy（太和）—— 可视化多智能体操作系统

> 黑客松参赛 Pitch

---

## 一句话 Elevator Pitch

Syntropy 是一个可视化多智能体操作系统，将 AI 黑盒转化为古代朝廷式的 2D 像素沙盘——让 Agent 的思考链、任务流转与记忆回溯在数字宫殿中一目了然。

---

## 问题陈述

当前 AI Agent 从炫酷 demo 到可靠生产环境之间存在巨大鸿沟：黑盒决策不可观测、多 Agent 调度缺乏结构化协议、工具调用风险不可控、记忆检索精度低。开发者像是在黑暗中指挥一支看不见的队伍。

---

## 解决方案

Syntropy 以 **"Harness（驾驭）"** 为核心理念——不是让 Agent 更聪明，而是让它们的协作可被人类 **看见、理解、干预**。

通过结构化 ACP 调度协议、风险分级审批流、RRF 混合记忆检索和全链路 Tracer，我们为多 Agent 系统装上方向盘、仪表盘和刹车。

---

## 核心差异化

| 现有方案 | Syntropy |
|---------|----------|
| LangGraph：工作流引擎 | **操作系统**——可视化运行态而非静态编排 |
| AutoGen：对话式多 Agent | **结构化任务调度** + 风险治理 |
| Dify：低代码应用构建器 | **面向开发者的 Agent 基础设施** |

我们不是另一个编排框架，而是 Agent 世界的"任务管理器 + 资源监视器"。

---

## 技术亮点

1. **结构化 ACP 调度协议**
   基于 JSON Schema 的零损耗任务分派，原生支持并行/串行/条件分支，替代自由文本指令的模糊性。

2. **RRF 混合记忆检索**
   FTS5 全文 + Vector 向量双路召回，加权融合实现精准记忆回溯，解决单一检索的召回偏差。

3. **风险分级 + 御批审批流**
   工具调用按风险等级自动分级，高风险操作触发人工审批，从 demo 迈向 production 的安全闸。

4. **全链路 Tracer 可观测性**
   从 LLM 请求到工具调用到 Agent 流转，每个节点可追踪、可回放、可调试。

5. **LLM 主动记忆捕获**
   Agent 运行时自动识别关键信息并持久化，无需人工标记，记忆随使用自我进化。

## Demo 场景：修建水利工程（3 分钟演示）

**丞相（Orchestrator）** 接到"修建水利工程"圣旨，秒级拆解并并行派发：

- **户部** —— 核算预算与粮草，调用历史账本检索
- **工部** —— 工程规划与施工方案，调用知识库检索先例
- **兵部** —— 征调民夫与兵力部署，检查权限与风险等级

**冲突发生**：户部发现预算超支 30%，触发 **黄色风险拦截**，丞相收到御批提醒。

**干预决策**：丞相调阅工部方案，发现可优化施工路线，重新派发户部复核。

**结果聚合**：三部委结果经结构化 ACP 协议汇聚，丞相生成最终奏折——**全程在 Phaser 2D 沙盘中实时可视化**，Agent 移动、思考链闪烁、审批流高亮。
