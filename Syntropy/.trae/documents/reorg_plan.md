# 朝廷组织架构重组计划

陛下，微臣已收到您的旨意。为了让天命系统更符合大统，微臣将对朝廷的组织架构进行大刀阔斧的改革。

## 1. 核心架构调整
新的组织架构将由以下人员组成：
*   **皇帝 (The Emperor)**：九五之尊，发布诏令。
*   **丞相 (The Prime Minister)**：统领百官，负责规划与任务分发（原中书省职能 + 部分尚书省职能）。
*   **六部 (The Six Ministries)**：各司其职，负责具体执行。
    *   **吏部 (Personnel)**：管理人事与官员状态。
    *   **户部 (Revenue)**：管理国库、数据与算力。
    *   **礼部 (Rites)**：负责对外交流（API）与祭祀（新闻/天气）。
    *   **兵部 (War)**：负责系统部署、运维与安全。
    *   **刑部 (Justice)**：负责审核（Code Review）、测试与审计（原门下省职能）。
    *   **工部 (Works)**：负责前端界面与基础设施建设。
*   **史官 (The Historian)**：负责记录日志与系统历史（Data Persistence）。

## 2. 代码重构步骤

### 2.1 更新 Agent 定义 (`useAgentStore.ts`)
*   修改 `AgentId` 类型，移除旧的 `censor`、`general` 等，新增六部尚书与史官。
    *   `emperor` (皇帝)
    *   `minister` (丞相)
    *   `official_personnel` (吏部尚书)
    *   `official_revenue` (户部尚书)
    *   `official_rites` (礼部尚书)
    *   `official_war` (兵部尚书)
    *   `official_justice` (刑部尚书)
    *   `official_works` (工部尚书)
    *   `historian` (史官)

### 2.2 更新场景布局 (`MainScene.ts`)
*   **重新排布工位**：
    *   皇帝：居中（太和殿）。
    *   丞相：皇帝侧前方。
    *   六部：分列两侧（左三部：吏、户、礼；右三部：兵、刑、工）。
    *   史官：角落（默默记录）。
*   **调整初始化逻辑**：确保新角色的 Sprite 和初始位置正确生成。

### 2.3 更新业务逻辑 (`CourtService.ts` & `roles/`)
*   **职能迁移**：
    *   `Secretariat` (中书省) -> 并入丞相职责。
    *   `Chancellery` (门下省) -> 移交刑部 (Justice) 负责审核。
    *   `DepartmentOfState` (尚书省) -> 简化为丞相的分发逻辑。
*   **六部具体化**：确保每个部门都有对应的处理逻辑（Mock 或 LLM 调用）。
*   **史官系统**：将日志记录功能具象化为史官的行动（如移动到书架记录）。

### 2.4 界面适配 (`ConsoleLayout` 等)
*   更新部门停靠栏 (`DepartmentDock`)，显示六部图标。
*   更新官员列表面板。

## 3. 预期效果
*   **更清晰的职责划分**：六部对应现代软件工程的各个环节（产品、后端、运营、测试、前端等）。
*   **更威严的朝堂**：文武百官分列两旁，更有仪式感。
