# 全新 UI 设计方案：太和殿指挥舱 (The Taihe Command Center)

用户反馈当前的 Bento Grid 布局“太丑”、“风格不符”、“位置不好”。我们将对其进行彻底的视觉升级，从“网页仪表盘”转型为**沉浸式游戏化指挥舱**。

## 核心设计理念

1. **去网页化 (De-webification)**: 移除所有像网页导航栏、面包屑、卡片间隙的设计。
2. **沉浸式 HUD (Immersive HUD)**: 顶部栏不再是导航，而是**抬头显示器 (HUD)**，实时监控六部状态。
3. **中式赛博美学 (Chinese Cyberpunk Aesthetic)**: 统一使用红/黑/金配色，结合像素风与现代玻璃态 (Glassmorphism)。

## 1. 组件重构计划

### A. 通用容器 (`PanelFrame`)

创建一个统一的面板容器组件，替代原本散乱的 `div`。

* **视觉**: 深色磨砂玻璃背景 + 金色细边框 + 四角回纹装饰。

* **功能**: 内置标题栏 (Header) 和 内容区域 (Content)。

* **目标**: 让左、中、右三个区域在视觉上统一。

### B. 顶部指挥台 (`ConsoleLayout` 重构)

原本的 `ConsoleLayout` 包含了导航按钮，现在需要改为纯粹的状态监控。

* **左侧**: **六部直值 (Ministry Status)**。

  * 移除 `DepartmentDock` 中的导航按钮。

  * 将“官员令牌”横向排列，作为常驻的状态指示灯（类似游戏的队友状态栏）。

  * 保留“上朝/退朝”按钮，作为核心功能开关。

* **中央**: **圣旨颁布 (Decree Pipeline)**。

  * 保持居中，作为视觉焦点。

  * 增加“卷轴展开”的视觉背景。

* **右侧**: **国运指标 (Metrics Panel)**。

  * 保持现状，优化微调。

### C. 布局结构 (`DashboardLayout` 调整)

* **Grid 比例**: 调整为 `20% (左) - 60% (中) - 20% (右)`，突出中央游戏画面。

* **无缝拼接**: 减小 Grid Gap，或者使用边框重叠设计，使其看起来像一个整体的控制台屏幕。

* **中央区域**:

  * `GameContainer` 将获得最隆重的“龙纹边框”。

  * `JailModal` 保持覆盖层地位。

## 2. 实施步骤

1. **创建** **`PanelFrame.tsx`**:

   * 实现带有中式边框装饰的容器组件。

   * 支持 `title`, `subtitle`, `action` 等属性。

2. **重构** **`ConsoleLayout.tsx`**:

   * 拆解 `DepartmentDock`，提取官员状态图标逻辑。

   * 重新布局为 HUD 模式。

3. **更新** **`DashboardLayout.tsx`**:

   * 应用 `PanelFrame`。

   * 调整 Grid 布局参数。

   * 移除旧的 `SectionHeader`，改用 `PanelFrame` 的内置 Header。

4. **样式微调**:

   * 确保 `z-index` 正确，避免覆盖。

   * 统一字体和颜色规范。

## 3. 预期效果

* 用户不再感觉到“网页感”，而是觉得在操作一个复杂的“天命系统”终端。

* 顶部导航栏不再显得多余，而是实用的状态监控。

* 整体风格高度统一。
