# 修复布局与缩放问题计划

用户反馈游戏画面“完全乱套了”。经排查，这是因为之前启用了 `RESIZE` 模式（根据窗口大小改变游戏逻辑坐标），但 `CourtService` 中的坐标逻辑（如议政厅位置 `x: 400`）是基于固定的 `800x600` 分辨率设计的。这导致画面元素错位、比例失调。

我们将采取以下措施修复此问题，确保游戏在任何屏幕尺寸下都能正确显示：

## 1. 恢复固定分辨率 (`GameContainer.tsx`)
- 将 `scale.mode` 改回 `Phaser.Scale.FIT`。
- 设置固定的内部逻辑分辨率：宽 `800`，高 `600`。
- 保留 `pixelArt: true` 以确保像素画风清晰。
- **效果**：游戏将始终以 `800x600` 的逻辑运行，自动缩放以适应浏览器窗口，不再拉伸变形或错位。

## 2. 规范化场景逻辑 (`MainScene.ts`)
- 移除动态监听 `resize` 事件的代码（不再需要）。
- 确保背景图强制缩放至覆盖 `800x600` 区域。
- 确认 `createDesk` 和障碍物生成逻辑使用固定的 `800x600` 坐标系。
- 移除场景创建中所有对 `window.innerWidth` 的依赖。

## 3. 验证坐标系 (`CourtService.ts`)
- 当前 `CourtService` 中的坐标（如 `COUNCIL: { x: 400, y: 150 }`）正是基于 `800` 宽度的中心点设计。
- 恢复固定分辨率后，这些逻辑坐标将再次准确对应画面位置。

## 4. 视觉微调
- 确保交互层（地板点击区域）大小严格匹配 `800x600`。
- 检查 Agent 的缩放比例（目前为 1.5 倍），在 `800x600` 分辨率下应大小适中。

通过以上调整，游戏逻辑将回归一致性，同时保持良好的跨设备适应性。
