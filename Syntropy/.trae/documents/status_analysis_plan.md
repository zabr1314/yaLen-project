# 天命系统当前进程分析计划

## 1. 目标
通过分析项目结构、参考项目 (`openclaw`, `openclaw-studio`)、最新文档及核心代码，精准定位当前项目的开发进程与状态。

## 2. 分析步骤

### 2.1 参考项目分析
- [ ] **OpenClaw (`openclaw/`)**: 确认其作为 Agent Runtime 的角色，以及它提供的 API/WebSocket 接口定义，评估其与主项目的集成准备度。
- [ ] **OpenClaw Studio (`openclaw-studio/`)**: 分析其功能定位（可能是旧版控制台或参考 UI），提取可复用的设计模式或逻辑。

### 2.2 核心文档与架构审查
- [ ] **重组计划 (`.trae/documents/reorg_plan.md`)**: 深入理解“三省六部”向“丞相+六部”转型的具体要求。
- [ ] **集成文档 (`OPENCLAW_INTEGRATION.md`)**: 确认 WebSocket 通信架构的设计与实施状态。
- [ ] **架构文档 (`docs/`)**: 把握整体设计理念（像素风朝堂 + 游戏引擎渲染）。

### 2.3 代码实现状态核查 (Gap Analysis)
- [ ] **数据层 (`src/store/useAgentStore.ts`)**: 检查 `AgentId` 和状态定义是否已完成向新架构的迁移。
- [ ] **表现层 (`src/game/MainScene.ts`)**: 验证角色初始化、场景布局、资源加载是否符合“丞相+六部”的新规制。
- [ ] **集成层 (`scripts/openclaw-connector.js`, `server/`)**: 检查 WebSocket 连接器和中转服务的代码框架是否已就绪。

## 3. 输出成果
- **当前进程定位**: 明确项目处于“架构重构”、“功能开发”还是“集成测试”阶段。
- **差异分析**: 列出文档规划与代码实现之间的具体差异（如：皮肤缺失、逻辑 Mock 等）。
- **下一步建议**: 基于当前状态，提出接下来的开发重点（如：完善六部逻辑、替换美术资源、联调 OpenClaw 等）。
