# 奏折阁与起居注功能分层与 UI 优化计划

## 1. 核心目标
1.  **功能区分**: 明确“奏折阁”与“起居注”的定位，避免信息冗余。
    *   **奏折阁 (Memorials Panel)**: 核心定位为 **“任务详情与对话存档”**。展示完整的任务内容、执行计划、以及皇帝与丞相的完整对话记录。
    *   **起居注 (Log Sidebar)**: 核心定位为 **“系统动态与行为监控”**。只展示关键事件的摘要（如“开始执行”、“位置移动”、“完成任务”），**不再展示**大段的对话内容，保持清爽。
2.  **UI 修复**: 解决奏折阁中长文本导致的显示溢出问题。

## 2. 实施方案

### 2.1 起居注瘦身 (`LogSidebar.tsx`)
- [ ] **内容过滤**:
    - 当日志类型为 `回复` (Reply) 时，不再渲染具体的 `details` 文本。
    - 改为显示简短的摘要，例如：“【回复】(点击奏折阁查看详情)”。
- [ ] **视觉降噪**: 减少气泡的高度，使其更像是一个紧凑的时间轴列表。

### 2.2 奏折阁优化 (`MemorialsPanel.tsx`)
- [ ] **长文本处理**:
    - 为每个奏折卡片中的“对话区域” (`ConversationView`) 添加 **最大高度限制** (e.g., `max-h-[400px]`)。
    - 启用 **内部滚动** (`overflow-y-auto`)。这样即使对话再长，也不会把整个卡片撑得无限大，导致看不见其他奏折。
- [ ] **自动滚动**: (可选) 当有新消息时，自动滚动到底部（暂不强制，优先解决溢出）。

## 3. 预期效果
- **起居注**: 变成一个干净的系统状态监视器，看着丞相忙忙碌碌，但不会被刷屏。
- **奏折阁**: 变成一个功能强大的聊天/任务历史记录板，单张卡片高度可控，内容可滚动查看。
