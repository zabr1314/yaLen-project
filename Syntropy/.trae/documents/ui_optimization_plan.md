# 天命系统 UI/UX 深度优化计划

## 核心目标
在已确立的“赛博古风”视觉基础上，进一步增强**沉浸感**、**交互反馈**和**细节质感**，使系统不仅仅是一个静态的 UI，而是一个有生命力的“天命中枢”。

## 优化方向与任务拆解

### 1. 视觉质感增强 (Visual Fidelity)
目前虽然有了木纹和金边，但整体材质感仍偏向扁平。
- **任务 1.1: 动态光影系统**
  - 为“御案”添加跟随鼠标微动的光泽效果（模拟烛光摇曳）。
  - 给金边和玉牌添加金属/玉石的高光反射。
- **任务 1.2: 粒子特效**
  - 在“灵火”燃烧时，增加向上飘散的微粒。
  - 在圣旨流转盖章时，增加朱砂飞溅的微小粒子。

### 2. 交互反馈优化 (Interaction & Feedback)
操作时的手感需要更加厚重、有仪式感。
- **任务 2.1: 音效反馈 (Sound FX)**
  - **拟旨输入**：打字时播放毛笔书写声（沙沙声）。
  - **盖章确认**：批红时播放沉重的印章敲击声。
  - **令牌互动**：鼠标悬停在令牌上时，播放玉石碰撞的清脆声。
  - **圣旨展开**：播放卷轴滚动声。
- **任务 2.2: 动效微调**
  - 优化 `DecreePipeline` 的展开动画，使其更像真实的卷轴滚动（带有弹性阻尼）。
  - 令牌点击时增加“按压”或“拿起”的位移动画。

### 3. 信息展示优化 (Information Architecture)
- **任务 3.1: 奏折详情预览**
  - 在 `LogSidebar`（文书档案）中，不仅显示文字，尝试将每条日志渲染成迷你的“奏折”卡片，点击可展开查看详细“批注”。
- **任务 3.2: 国运指标可视化**
  - 将 CPU/Users 等数字指标，配合动态的“水位”或“星盘”可视化（例如：CPU 占用高时，星盘转速变快，或水位上涨）。

### 4. 沉浸式彩蛋 (Immersive Details)
- **任务 4.1: 闲置动作 (Idle Animations)**
  - 当无操作时，Agent（令牌）可以有呼吸灯效果，或者偶尔“震动”一下表示正在思考。
  - 御案上可以随机出现一些装饰性物件（如：茶杯冒热气、瑞兽香炉飘烟）。

## 执行计划 (第一阶段：视觉与动效)

1.  **优化 `DepartmentDock`**: 为令牌添加鼠标悬停的光影流光效果，增强玉石质感。
2.  **优化 `DecreePipeline`**: 改进圣旨卷轴的 CSS 材质，增加纸张纹理的粗糙感；优化盖章动画的爆发力。
3.  **优化 `ConsoleLayout`**: 在御案两侧添加微弱的动态烛光（CSS 动画）。
4.  **优化 `MetricsPanel`**: 将数字变化改为滚动计数器（类似旧式机械计数器或翻牌器），增加复古机械感。

## 后续建议 (待定)
- 引入 Web Audio API 实现音效系统（需确认是否允许播放声音）。
- 引入 Canvas/WebGL 实现更复杂的粒子背景（如：龙气缭绕）。
