# AI 帝国·数字中枢 MVP 视觉增强计划 (Sprint 1.5)

目前的基础演示确实较为简陋，仅有基本的移动和日志功能。为了让 Demo 更有“天命系统”的代入感，我们需要增强视觉表现力、丰富环境细节，并初步引入“天牢”概念。

## 核心目标
1.  **视觉升级**：从简单的几何图形升级为更具象的像素风格（程序化生成），让角色看起来更像“人”。
2.  **环境细节**：增加办公桌细节（电脑、文件），划定“天牢”警戒区。
3.  **交互增强**：实现头顶气泡对话（Bubble Text），让 Agent 的思考过程可视化。
4.  **状态反馈**：增加“工作中”、“思考中”、“报错”等状态的视觉反馈。

## 详细实施步骤

### 1. 角色形象升级 (Visual Upgrade)
-   **目标**：不再是单调的圆点，而是有头有身体的像素小人。
-   **实现**：
    -   使用 `Phaser.Graphics` 绘制更复杂的纹理：一个圆形头部 + 一个矩形身体。
    -   为不同角色的职业（丞相、工部）设计不同的颜色/服饰特征（如丞相带官帽/长袍，工部穿格子衫/短袖）。
    -   添加简单的“呼吸”动画（Scale 缩放）让角色看起来是活的。

### 2. 办公室环境装修 (Office Polish)
-   **目标**：让地图看起来像一个真正的“数字中枢”。
-   **实现**：
    -   **办公桌**：在桌子上增加“电脑显示器”（发光矩形）和“文件堆”（白色小块）。
    -   **装饰**：添加几盆绿植（绿色圆形+叶子）。
    -   **地板**：绘制网格线或瓷砖纹理，增加空间感。
    -   **天牢区域**：在地图角落绘制带有黄黑警示条纹的区域，作为“异常拦截区”。

### 3. 头顶气泡对话系统 (Bubble System)
-   **目标**：可视化 Agent 的思维活动。
-   **实现**：
    -   创建一个 `SpeechBubble` 类（Container），跟随 Agent 移动。
    -   当 Agent 状态改变或移动时，显示对应的文字（如“前往工位...”、“正在请求 API...”、“⚠️ 异常！”）。
    -   支持自动消失和动态文本。

### 4. 交互与控制台增强 (Interaction & Console)
-   **目标**：让操作更有反馈感。
-   **实现**：
    -   **控制台**：在 Sidebar 增加手动触发“工作”和“报错”的按钮。
        -   “派活”按钮：Agent 移动到办公桌，头顶冒出“正在工作...”，电脑屏幕变色。
        -   “注入异常”按钮：Agent 头顶冒出“❌ 越权访问”，并被强制移动到“天牢”区域。
    -   **点击反馈**：点击 Agent 时，在 Sidebar 显示其详细属性（ID、当前任务、状态）。

## 预期效果
完成以上改进后，Demo 将不再是“红点移动”，而是一个拥有像素小人、会思考（冒泡）、有工作区和惩罚区（天牢）的微型数字社会。这能更好地向用户传达“可视化治理”的核心理念。
