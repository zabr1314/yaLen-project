# 计划：交互式沙盘与动态环境反馈 (优化版)

## 目标
根据您的反馈，我认为“拖拽奏折”的交互可能过于单一且操作不便。因此，我将方案升级为 **“角色直控 (Direct Agent Control)”** 与 **“场景化交互 (Zone-based Interaction)”**。

核心理念：**像玩 RPG 游戏一样管理你的 Agent。**

## 拟定变更

### 1. 交互升级：角色上下文菜单 (Context Menu)
- **文件**: `src/game/MainScene.ts`
- **痛点解决**: 避免繁琐的拖拽，改为更直观的点击操作。
- **实现逻辑**:
    -   **右键点击 (Right Click)** 任意 Agent，弹出一个环形或列表式菜单。
    -   **菜单选项**:
        -   `🗣️ 下达旨意`: 自动聚焦控制台输入框，并填充 `@AgentName`。
        -   `🔍 查看记忆`: 直接打开“记忆透视镜”面板，查看该 Agent 的 RAG 状态。
        -   `💤 强制休眠`: 手动触发 `onSleep`，强制进行记忆压缩。
        -   `🛑 关入天牢`: (调试用) 手动触发风控拦截演示。

### 2. 交互升级：区域拖拽 (Zone Dragging)
- **文件**: `src/game/MainScene.ts`
- **痛点解决**: 提供一种“物理化”的管理方式。
- **实现逻辑**:
    -   允许用户 **直接拖拽 Agent 本身**。
    -   **定义功能区域 (Zones)**:
        -   **太和殿 (Meeting Area)**: 拖拽至此，强制该 Agent 进入“上朝”状态。
        -   **天牢 (Jail)**: 拖拽至此，强制挂起 Agent 任务。
        -   **御书房 (Private Study)**: 拖拽至此，开启一对一私密对话模式（屏蔽其他 Agent 消息）。

### 3. 环境反馈：动态天气与情绪氛围
- **文件**: `src/game/MainScene.ts`
- **实现逻辑**:
    -   **情绪氛围 (Emotional Ambience)**:
        -   当 Agent 普遍处于 `ERROR` 或 `WAITING` 状态时，场景色调变暗冷（蓝色滤镜），模拟“朝廷气氛压抑”。
        -   当任务高效完成时，场景色调变暖亮（金色滤镜），模拟“盛世”。
    -   **昼夜系统**: 保留原计划的昼夜交替，增加沉浸感。

## 验证计划
1.  **菜单测试**: 右键点击丞相，确认能否弹出菜单并触发“查看记忆”面板。
2.  **拖拽测试**: 将“兵部尚书”拖入“天牢”区域，验证是否触发挂起状态。
3.  **氛围测试**: 模拟多个 Agent 报错，观察场景光照是否随之变化。
