# 计划：交互式沙盘与动态环境反馈

## 目标

将 2D 沙盘从被动的“展示屏”升级为主动的“控制台”。

1. **拖拽指派 (Drag & Drop)**: 允许用户直接将“奏折”（任务图标）拖拽到 Agent 身上，快速触发交互或指派任务。
2. **动态环境**: 增加 Agent 工作负载的视觉反馈（火花/烟雾）以及昼夜交替系统，增强沉浸感。

## 现状分析

* **MainScene.ts**: 目前仅支持点击地面移动皇帝 (`floor.on('pointerdown')`)，缺乏与 Agent 的直接交互。

* **Agent 可视化**: 仅有空闲/行走动画，缺乏“高负载”或“思考中”的视觉提示。

* **环境**: 背景是静态的，缺乏时间流逝感。

## 拟定变更

### 1. 拖拽任务指派 (前端)

* **文件**: `src/game/MainScene.ts`

* **实现逻辑**:

  * 创建一个可拖拽的“奏折”图标生成器。

  * 实现 `dragend` 逻辑：

    * 检测释放位置是否在某个 Agent 的碰撞体内。

    * 如果是，触发 `task_assignment` 事件（模拟触发，或直接在控制台输入框填充 `@AgentName`）。

    * **视觉反馈**: 当拖拽物悬停在 Agent 上方时，Agent 高亮显示。

### 2. "忙碌"状态特效

* **文件**: `src/game/MainScene.ts` & `src/game/Agent.ts`

* **实现逻辑**:

  * 当 Agent 处于 `WORKING` 或 `THINKING` 状态时，在其办公桌位置生成粒子特效。

  * **过载特效 (Overload)**: 模拟 Agent 处理复杂任务（如 RAG 检索），从办公桌冒出微小的“烟雾”或“火花”粒子。

### 3. 昼夜交替系统

* **文件**: `src/game/MainScene.ts`

* **实现逻辑**:

  * 添加一个全屏的矩形遮罩 (`Rectangle`)，颜色为深蓝 (`0x000033`)。

  * 使用 `tween` 根据时间（或按键调试）调整透明度：

    * **白昼**: Alpha 0。

    * **黑夜**: Alpha 0.5 (半透明黑暗)。

  * **灯光系统**: 在每张办公桌旁添加“灯笼”光源（圆形渐变纹理），仅在黑夜模式下开启。

## 验证计划

1. **交互测试**: 拖拽任务图标到“丞相”身上，验证控制台是否自动填充 `@minister`。
2. **视觉测试**: 手动设置 Agent 状态为 `working`，观察是否有粒子特效。
3. **时间测试**: 通过调试按键（如 'N'）切换昼夜，验证灯光和遮罩效果。
