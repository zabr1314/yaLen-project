# 天命系统深度优化方案

基于当前的系统状态和竞品分析，我为你梳理了三个可以进一步提升体验和健壮性的方向。

## 方向一：沉浸式视觉增强 (Visual Immersion) - **推荐**
目前的 `error` 和 `offline` 状态在视觉上反馈较弱。为了强化“天命系统”的隐喻，我们可以引入更具戏剧性的表现。

### 1. 天牢系统 (Heaven's Prison)
*   **触发**: 当 Agent 状态为 `error` 时。
*   **表现**: Agent 不再只是头顶冒红气泡，而是被“打入天牢”（移动到特定的监狱区域，并加上栅栏特效）。
*   **操作**: 皇帝（用户）需要点击“赦免”按钮，Agent 才能恢复自由（重置为 `idle`）。

### 2. 离魂模式 (Soul Mode)
*   **触发**: 当 Agent 状态为 `offline` 时。
*   **表现**: Agent 的 Sprite 变为半透明（Ghost），并飘荡回休息区，表示“魂魄离体，等待归位”。

## 方向二：服务端数据持久化 (Persistence)
目前的 Relay Server 是纯内存存储 (`const agents = {}`)，重启服务会导致所有已注册的 Agent 丢失，直到它们下一次上报。

*   **改进**: 在 `server/index.js` 中增加文件读写逻辑。
*   **机制**: 每次状态更新时异步写入 `agents-state.json`；服务启动时读取该文件恢复现场。

## 方向三：指令确认机制 (Command ACK)
目前的 `sendCommand` 是“发后即忘 (Fire-and-Forget)”，前端无法确认 Agent 是否真的收到了指令。

*   **改进**: 升级 WebSocket 协议。
*   **流程**: 前端 `emit('command')` -> Server 转发 -> Connector 收到后 `emit('command_ack')` -> Server 转发 -> 前端显示“绿勾”。

## 综合建议
鉴于这是一个**可视化管理平台**，我强烈建议优先实施 **方向一 (沉浸式视觉增强)**，特别是“天牢”系统，这会让错误处理变得非常有趣且符合直觉。
