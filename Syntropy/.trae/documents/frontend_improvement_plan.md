# 前端优化建议与计划 (中文汉化版)

## 1. 现状分析
在接入了新的后端 (`server/index.js`) 和 OpenClaw 连接器后，前端的核心逻辑已从纯模拟转变为混合驱动模式。目前存在以下几个明显的缺失点或可优化空间：

### A. 角色完整性缺失
- **吏部 (Ministry of Personnel)**: 在 `CourtService` 中缺少对应的逻辑类，导致如果任务被分派给吏部，系统可能会报错或无法执行。虽然 ID 映射里有 `official_personnel`，但业务逻辑层缺失。
- **角色差异化**: 目前所有官员的行为逻辑趋同（都是走到工位 -> 等待 -> 完成），缺乏不同部门的特色动画或交互（例如兵部应该在巡逻，户部应该在算账）。

### B. 实时反馈与错误处理
- **连接状态感知**: 用户不知道是否成功连接到了后端，或者后端是否连接到了 OpenClaw。如果 WebSocket/HTTP 轮询失败，前端只是静默失败，没有 UI 提示。
- **日志可视化的局限**: 目前的日志主要服务于内阁模式。在 Live 模式下，OpenClaw 传回的 `current_task` 虽然显示在气泡里，但没有持久化到侧边栏日志中，导致用户无法回溯 Agent 做了什么。

### C. 交互与控制
- **模式切换**: 目前是代码写死的自动启动 Live 模式。用户无法在界面上主动切换 "模拟模式" 和 "实时模式"。
- **干预能力**: 用户无法暂停 Agent 的行动，或者强制召回某个 Agent。

## 2. 优化计划 (汉化优先)

### 第一阶段：补全核心缺失与汉化 (优先级：高)
1.  **实现 `MinistryOfPersonnel`**: 补全吏部的业务逻辑类，确保六部体系完整。(已完成)
2.  **UI 全面汉化**: 确保界面上所有英文文本（如 "Live Mode", "Offline", "Status", "Log" 等）全部替换为古风或现代中文（如 "实时模式", "离线", "状态", "奏折" 等）。
3.  **增强连接状态提示**: 在 UI 顶部添加一个状态指示器 (如 "🟢 实时连接" / "🔴 连接断开")，显示与 Relay Server 的连接健康度。

### 第二阶段：提升可视化体验 (优先级：中)
1.  **日志双向同步**: 将 LiveAgentService 获取到的状态变更（特别是从 working -> idle 的完成事件）写入到 `useCourtStore` 的日志系统中，让右侧日志栏也能反映真实 Agent 的工作历史。
2.  **特色动作**: 为不同职业的 Agent 添加微小的 idle 动画差异（如果 Phaser 资源允许）。

### 第三阶段：增强控制力 (优先级：低)
1.  **控制台开关**: 在界面上添加 "连接/断开" 按钮。
2.  **强制召回**: 点击某个 Agent 可以将其强制状态重置为 Idle（发送指令给后端，如果后端支持的话；或者仅前端重置）。
